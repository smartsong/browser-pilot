## 一、安装步骤

### 1. 加载扩展

1. 打开 Edge 浏览器，地址栏输入：`edge://extensions/`
2. 右上角开启 **"开发人员模式"**
3. 点击 **"加载已解压的扩展程序"**
4. 选择目录：`E:\QClaw\workspace\projects\browser-pilot\extension`
5. 扩展加载成功后，会看到 **"Browser Pilot"** 出现在扩展列表

### 2. 验证注入

1. 打开任意网页（如 `https://www.baidu.com`）
2. 按 `F12` 打开开发者工具 → 切到 **控制台（Console）**
3. 看到这条日志说明注入成功：
   ```
   🤖 [Browser Pilot v1.0] 标签页已注册: tab_xxxx
   ```

### 3. 启动服务器

在 PowerShell 中运行：
```powershell
cd E:\QClaw\workspace\projects\browser-pilot
python server.py
```

看到以下输出说明服务器启动成功：
```
🤖 [Browser Pilot Server v0.3] 启动在 0.0.0.0:19789
```

## 二、架构说明

### 组件关系

```
AI Agent (你)
    ↓ HTTP API
server.py (中间服务器)
    ↓ 轮询 (poll)
content.js (注入到网页的脚本)
    ↓ 操作
浏览器页面 (DOM)
```

### 通信流程

1. **AI Agent** 发送指令到 `server.py` (`/command`)
2. **server.py** 存储任务到 `tasks[]`
3. **content.js** 每 2 秒轮询 `/poll`，取到任务
4. **content.js** 执行指令（操作 DOM）
5. **content.js** 提交结果到 `/result`
6. **AI Agent** 查询 `/result?task_id=xxx` 拿到结果

## 三、API 说明

### 1. 提交指令

**请求：**
```http
POST /command
Content-Type: application/json

{
  "command": {
    "action": "get_title"
  }
}
```

**响应：**
```json
{
  "task_id": "abc123",
  "status": "pending"
}
```

### 2. 查询结果

**请求：**
```http
GET /result?task_id=abc123
```

**响应：**
```json
{
  "success": true,
  "data": {
    "title": "百度一下，你就知道"
  }
}
```

### 3. 注册（自动，无需手动调用）

**请求：**
```http
GET /register?tab_id=tab_xxxx&title=页面标题&url=https://...
```

### DOM 操作指令

#### `click` — 点击元素

**参数：**
- `selector` (可选)：CSS 选择器
- `text` (可选)：按文本内容查找

**请求：**
```json
{
  "command": {
    "action": "click",
    "params": {
      "selector": "#su"
    }
  }
}
```

**或使用文本内容查找：**
```json
{
  "command": {
    "action": "click",
    "params": {
      "text": "百度一下"
    }
  }
}
```

**响应：**
```json
{
  "success": true,
  "data": {
    "clicked": "#su",
    "tag": "INPUT"
  }
}
```

---

#### `input` — 填入文本（立即完成）

**参数：**
- `selector` (必填)：CSS 选择器
- `value` (必填)：要填入的文本
- `clear` (可选)：是否清空原有内容，默认 `true`

**请求：**
```json
{
  "command": {
    "action": "input",
    "params": {
      "selector": "#kw",
      "value": "Browser Pilot",
      "clear": true
    }
  }
}
```

**响应：**
```json
{
  "success": true,
  "data": {
    "input": "#kw",
    "value": "Browser Pilot"
  }
}
```

---

#### `type` — 逐字输入（模拟人工）

**参数：**
- `selector` (必填)：CSS 选择器
- `text` (必填)：要输入的文本

**请求：**
```json
{
  "command": {
    "action": "type",
    "params": {
      "selector": "#kw",
      "text": "测试输入"
    }
  }
}
```

**响应：**
```json
{
  "success": true,
  "data": {
    "typed": "#kw",
    "text_length": 4
  }
}
```

---

#### `extract` — 提取页面内容

**参数：**
- `selector` (可选)：CSS 选择器（不填则提取整个页面）
- `attr` (可选)：提取指定属性（如 `href`, `src`）
- `multiple` (可选)：是否提取多个元素，默认 `true`

**请求（提取整个页面）：**
```json
{
  "command": {
    "action": "extract"
  }
}
```

**请求（提取指定元素）：**
```json
{
  "command": {
    "action": "extract",
    "params": {
      "selector": "#lg",
      "multiple": false
    }
  }
}
```

**响应（单个元素）：**
```json
{
  "success": true,
  "data": {
    "text": "百度Logo",
    "html": "<div id=\"lg\">...</div>",
    "attr": null
  }
}
```

**响应（多个元素）：**
```json
{
  "success": true,
  "data": {
    "count": 5,
    "items": [
      { "text": "...", "html": "...", "attr": null },
      ...
    ]
  }
}
```

---

#### `scroll` — 滚动页面

**参数：**
- `direction` (可选)：滚动方向，`down` / `up` / `top` / `bottom`，默认 `down`
- `amount` (可选)：滚动像素数，默认 `500`

**请求：**
```json
{
  "command": {
    "action": "scroll",
    "params": {
      "direction": "down",
      "amount": 500
    }
  }
}
```

**响应：**
```json
{
  "success": true,
  "data": {
    "scrolled": "down",
    "amount": 500,
    "scrollY": 464
  }
}
```

### 高级指令

**参数：**
- `ms` (可选)：等待毫秒数，默认 `1000`

**请求：**
```json
{
  "command": {
    "action": "wait",
    "params": {
      "ms": 3000
    }
  }
}
```

**响应：**
```json
{
  "success": true,
  "data": {
    "waited_ms": 3000
  }

```

---

#### `execute_js` — 执行自定义 JS（⚠️ 有限制）

**参数：**
- `code` (必填)：要执行的 JS 代码（必须是 `return` 语句）

**请求：**
```json
{
  "command": {
    "action": "execute_js",
    "params": {
      "code": "return document.title"
    }
  }
}
```

**响应（成功）：**
```json
{
  "success": true,
  "data": {
    "result": "百度一下，你就知道"
  }
}
```

**响应（失败 — CSP 限制）：**
```json
{
  "success": false,
  "message": "Evaluating a string as JavaScript violates the following Content Security Policy directive..."
}
```

**⚠️ 已知限制：**
- 某些网站（如百度）的 CSP 策略会阻止 `eval()` / `new Function()`
- 解决方案：需要修改 `content.js`，使用 `chrome.scripting.executeScript()` API（需调整架构）

## 五、完整工作流示例

### 场景：在百度搜索 "Browser Pilot" 并提取结果

**步骤1：导航到百度**
```powershell
Invoke-RestMethod -Uri "http://127.0.0.1:19789/command" `
  -Method POST `
  -Body '{"command":{"action":"navigate","params":{"url":"https://www.baidu.com"}}}' `
  -ContentType "application/json"
```

**步骤2：在搜索框输入文本**
```powershell
Invoke-RestMethod -Uri "http://127.0.0.1:19789/command" `
  -Method POST `
  -Body '{"command":{"action":"input","params":{"selector":"#kw","value":"Browser Pilot"}}}' `
  -ContentType "application/json"
```

**步骤3：点击搜索按钮**
```powershell
Invoke-RestMethod -Uri "http://127.0.0.1:19789/command" `
  -Method POST `
  -Body '{"command":{"action":"click","params":{"selector":"#su"}}}' `
  -ContentType "application/json"
```

**步骤4：等待页面加载**
```powershell
Invoke-RestMethod -Uri "http://127.0.0.1:19789/command" `
  -Method POST `
  -Body '{"command":{"action":"wait","params":{"ms":2000}}}' `
  -ContentType "application/json"
```

**步骤5：提取搜索结果**
```powershell
Invoke-RestMethod -Uri "http://127.0.0.1:19789/command" `
  -Method POST `
  -Body '{"command":{"action":"extract","params":{"selector":".result","multiple":true}}}' `
  -ContentType "application/json"
```

## 七、常见问题

### 1. 扩展加载失败："Couldn't load icon"

**原因：** 图标文件缺失或格式错误。

**解决：**
- 确保 `icons/` 目录存在且包含 `icon16.png`, `icon48.png`, `icon128.png`
- 或使用文档中的 Python 脚本生成简单图标

---

### 2. 控制台看不到 "🤖 [Browser Pilot] 内容脚本已加载" 日志

**原因：**
- `content_scripts` 的 `matches` 不匹配当前页面
- 扩展被禁用
- 脚本加载失败

**解决：**
1. 检查 `manifest.json` 的 `content_scripts.matches` 是否包含当前页面 URL
2. 打开 `edge://extensions/`，确认 **Browser Pilot** 已启用
3. 查看控制台是否有 JS 错误（红色日志）

---

### 3. 发送指令后一直拿不到结果

**原因：**
- `server.py` 没启动
- 扩展没有轮询到任务（网络问题）
- 任务执行失败（selector 不存在等）

**解决：**
1. 确认 `server.py` 在跑（看到 "启动在 0.0.0.0:19789" 日志）
2. 打开 F12 控制台，看扩展有没有收到任务日志：
   ```
   🤖 [BP] 执行: get_title
   ```
3. 查询结果时，如果 `success=false`，查看 `message` 字段的错误信息

---

### 4. `execute_js` 报错：CSP 限制

**原因：** 某些网站的 Content Security Policy 不允许执行动态 JS。

**解决（临时）：**
- 避免使用 `execute_js`，改用其他指令组合
- 或在 `chrome.scripting.executeScript()` 里执行（需修改 `background.js`）

**解决（根本）：**
- 修改架构，把 JS 执行放到 `background.js`，通过 `chrome.scripting.executeScript()` 调用

---

### 5. 页面跳转后扩展不工作

**原因：** 页面跳转后，`content.js` 会重新注入并执行 `register()`，应该能正常工作。

**验证：**
1. 跳转后按 F12，看有没有新的注册日志
2. 如果没有，可能是 `manifest.json` 的 `content_scripts.matches` 不匹配新页面 URL

## 八、已知限制

1. **`execute_js` 受 CSP 限制** —— 某些网站无法执行动态 JS
2. **`scroll` 在短页面无效** —— 页面内容不够长时，`scrollY` 始终为 0
3. **`extract` 对 `input` 元素无效** —— `textContent` 对输入框返回空，应该用 `value` 属性
4. **无错误重试机制** —— 任务失败后不会自动重试（需在 AI Agent 侧实现）
5. **只支持 HTTP** —— `server.py` 是 HTTP，生产环境需要 HTTPS + 认证
